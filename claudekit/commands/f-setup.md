# /f-setup — Figma Workflow Config Wizard

One-time setup that writes `.claude/figma.yaml` for the current project.
All f-* skills (`f-import`, `f-ui-kit`, `f-page`, `f-review`) read this file at startup.

---

## Step 0 — Check existing config

Read `.claude/figma.yaml` if it exists.

If found, show the current contents and ask:

```
.claude/figma.yaml already exists. What would you like to do?
[1] Overwrite (run setup again)
[2] Edit specific values
[3] Cancel
```

If [3]: stop.
If [2]: jump to the relevant step.

---

## Step 1 — Check Figma MCP availability

Try to determine if `mcp__figma__*` tools are available in the current session.

Ask the user:

```
Is the Figma MCP configured in this Claude Code session?
(If unsure: run `claude mcp list` in a terminal and check for a "figma" entry)

[1] Yes — MCP tools are available
[2] No — I'll use export folders only
```

Set `mcp_available` accordingly.

If [2]: note that hybrid mode and MCP-only mode won't be available.

---

## Step 2 — Detect framework

Inspect project root:

| File found | Framework |
|-----------|----------|
| `next.config.ts` or `next.config.js` | Detect router: `app/` dir → `nextjs-app`; `pages/` dir → `nextjs-pages` |
| `vite.config.ts` or `vite.config.js` | `vite` |
| `package.json` with `"react-scripts"` | `cra` |
| None | `auto` |

Show detected value. Ask user to confirm or override.

---

## Step 3 — Component output base

Default: `src/components`

Ask:
```
Where should UI kit components be placed?
Default: src/components  (UI kit lands at <base>/<module>/ui-kit/)
> [enter to accept or type path]
```

---

## Step 4 — Pages output base

Infer from framework:
- `nextjs-app` → `app`
- `nextjs-pages` → `pages`
- `vite` / `cra` → `src/pages`

Ask:
```
Where should generated pages be placed?
Default: <inferred>
> [enter to accept or type path]
```

---

## Step 5 — Design tokens path

Look for existing token files:

```bash
find . -name "tokens.css" -o -name "variables.css" -o -name "design-tokens.css" | grep -v node_modules | head -5
```

If found: suggest that path.
If not found: suggest `src/styles/tokens.css`.

Ask:
```
Where should design tokens (CSS vars) be written?
Default: <suggestion>
> [enter to accept or type path]
```

Also ask:
```
Write token values automatically on import? (or report diffs only)
[1] Yes — write tokens.css on each f-import run
[2] No — show diff only, I'll update manually
```

---

## Step 6 — Write `.claude/figma.yaml`

```yaml
# Generated by /f-setup — edit manually if needed
version: 1

mcp_available: <true|false>
figma_file_key: null

output:
  component_base: <component_base>
  pages_base: <pages_base>
  framework: <framework>

design_tokens:
  path: <tokens_path>
  write: <true|false>

manifest_dir: .figma-import
```

Create `.claude/` directory if it does not exist.

---

## Step 7 — Summary

```
## f-setup complete

Config written to .claude/figma.yaml

  MCP available:     yes / no
  Framework:         nextjs-app
  Components base:   src/components
  Pages base:        app
  Tokens path:       src/styles/tokens.css
  Tokens auto-write: yes

Next steps:
  /f-import <figma-url>              → import from Figma via MCP
  /f-import --export <folder-path>   → import from a Figma export folder
  /f-import --hybrid <url> <folder>  → MCP classification + export code
```
